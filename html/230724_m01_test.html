** Module과 Scope Test **

      * m01_bar.js : let 사용
      => window.menu : undefined ( 즉, 전역변수 아님)
      => menu : 출력 잘됨
        
      * m01_foo.js : 이전코드의 영향을 받음
      => window.menu : Uncaught SyntaxError: Identifier 'window.menu' has already...
      => menu : Uncaught SyntaxError: Identifier 'menu' has already been declared
        
      * 결론 : JS는 기본적으로 Scope 구분없이 전역으로 인식함
      1) let 사용시
        전역변수로 사용되므로 scope 적용 안되지만, 윈도우의 객체로 인식되지않음.
      2) var와 비교
        전역변수로 사용되므로 scope 적용 안되고, 윈도우의 객체로 인식됨.
        bar, foo 모두 출력(foo에서 값이 변경)
